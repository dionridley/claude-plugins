# Contributing to Project Management Plugin

Thank you for your interest in contributing to the Project Management Plugin for Claude Code!

## How to Contribute

### Reporting Issues

If you find a bug or have a feature request:

1. Check existing issues to avoid duplicates
2. Create a new issue with a clear title and description
3. Include steps to reproduce for bugs
4. Add relevant labels (bug, enhancement, documentation, etc.)

### Submitting Changes

1. **Fork the repository**

2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make your changes**
   - Follow existing code style and conventions
   - Update documentation if needed
   - Add tests if applicable

4. **Test your changes**
   - Run the plugin commands in a test project
   - Verify all existing functionality still works
   - Test edge cases

5. **Commit your changes**
   ```bash
   git commit -m "Add: brief description of changes"
   ```

   Commit message prefixes:
   - `Add:` - New features
   - `Fix:` - Bug fixes
   - `Update:` - Updates to existing features
   - `Docs:` - Documentation changes
   - `Refactor:` - Code refactoring

6. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```

7. **Create a Pull Request**
   - Provide a clear description of changes
   - Reference any related issues
   - Include testing steps

## Development Guidelines

### Command Files

When creating or modifying command files (`commands/*.md`):

- Use clear, descriptive frontmatter
- Include comprehensive instructions for Claude
- Document all phases and decision points
- Handle edge cases gracefully
- Provide helpful error messages

### Templates

When modifying templates (`templates/*.md`):

- Keep templates language-agnostic
- Use clear placeholder syntax
- Include helpful comments
- Maintain consistent formatting

### Documentation

- Update README.md for user-facing changes
- Update CHANGELOG.md for all notable changes
- Keep examples current and working

## Code of Conduct

### Our Standards

- Be respectful and inclusive
- Welcome newcomers and help them get started
- Accept constructive criticism gracefully
- Focus on what's best for the community

### Unacceptable Behavior

- Harassment or discrimination
- Trolling or insulting comments
- Publishing others' private information
- Other conduct inappropriate in a professional setting

## Questions?

If you have questions about contributing:

1. Check the README for usage information
2. Review existing issues and discussions
3. Create a new issue with your question

## Recognition

Contributors will be recognized in:
- CHANGELOG.md for significant contributions
- README.md acknowledgments section (for major contributors)

Thank you for helping improve the Project Management Plugin!
